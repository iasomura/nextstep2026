# STYLE_GUIDE — 論文表記ルール

対象: CSS論文「3段カスケード型フィッシング検出における投入制御とルール統合の効果分析」

---

## 1. 言語方針

**日本語本文 + 図も日本語**（CSS は日本語論文）

- 英語術語は **初出で英併記**、以降は日本語のみ
  - 例: 「投入率（call rate）」→ 以降「投入率」
- 表の列ヘッダは日本語を基本とし、略語が定着しているもの（TP/FP/TN/FN, F1, FPR, FNR）はそのまま使用可

---

## 2. 用語の正規形

| 正規形（日本語） | 英語（初出で併記） | 備考 |
|-----------------|------------------|------|
| Stage 1 / Stage 2 / Stage 3 | — | 半角スペース付き、常にこの表記 |
| 投入率 | call rate | Stage 3 投入率 = \|difficult subset\| / \|D_test\| |
| 自動判定 | auto-decision | Stage 1 または Stage 2 で確定した判定 |
| 自動判定誤り | auto-decision error | 自動判定のうち誤分類 |
| 低確信度領域 | gray zone | Stage 1 出力が自動判定閾値の範囲外 |
| 投入 | handoff | 後段へ送ること |
| 困難集合 | difficult subset | Stage 3 に到達した集合（n=11,952） |
| 誤り確率推定 | error probability estimation | Stage 2 の LR |
| 証明書ゲート | certificate gate | Stage 2 の hard gate |
| ルール発火 | rule firing | ルールが条件を満たして適用されたこと |
| 偽陰性 / 偽陽性 | FN / FP | 略語は英語のまま |

---

## 3. 数値表記ルール

| 項目 | ルール | 例 |
|------|--------|-----|
| 桁区切り | カンマ（3桁ごと） | 127,222 / 11,952 |
| パーセント | 半角%、直前スペースなし | 9.4% / 98.67% |
| 小数桁 | 重要数値は小数点以下2桁 | F1=98.67%, FPR=0.84% |
| 大量件数 | カンマ区切り、単位「件」は任意 | 60,767件 or 60,767 |
| 閾値 | ギリシャ文字 τ（tau） | τ=0.4 |
| 処理時間 | 秒（s）、小数2桁 | p50=8.31s |
| 分位点 | p50 / p90 / p99（小文字p） | p50=8.31s, p99=28.59s |

---

## 4. 図ラベル方針

| 要素 | ルール |
|------|--------|
| タイトル | 日本語。例:「Stage 3 投入率 vs 自動判定誤り」 |
| 軸ラベル | 日本語 + 単位。例:「処理時間（秒）」「累積割合」 |
| 凡例 | 日本語。例:「自動判定誤り件数」「誤り率（%）」 |
| 注記 | 日本語。例:「※ Stage 1+2 の自動判定分のみ」 |
| n 表記 | 図中に明示。例:「n=11,952」 |
| 運用点 | 記号（★等）+ 注記。例:「★ 運用点（τ=0.4）」 |
| フォント | 日本語: IPAexGothic / Noto Sans CJK、英数字: sans-serif |

---

## 5. 表の方針

| 要素 | ルール |
|------|--------|
| 列ヘッダ | 日本語を基本。TP/FP/TN/FN/F1/FPR/FNR は英略語 |
| キャプション | 日本語。母数を明記。例:「表5: Stage 3 性能（困難集合、n=11,952）」 |
| 数値 | 上記の数値表記ルールに従う |

---

## 6. 本文参照の書き方

| 対象 | 書き方 |
|------|--------|
| 図 | 図1、図2、…（全角「図」＋半角数字） |
| 表 | 表1、表2、… |
| 節 | §1.1、§4.2、…（§＋半角数字） |
| アルゴリズム | Algorithm 1（英語のまま） |

---

## 付記: 英語論文にする場合の差分

- 用語の正規形を英語に統一（括弧内の英語をそのまま使用）
- 図ラベル・キャプションを英語に変更
- 桁区切り・パーセント表記はそのまま
- 「図1」→「Fig. 1」、「表1」→「Table 1」
